Database Schema Summary

Main Entities

  • • Users (customers, providers, admins)
  • • Services (hotels, rides, events, flights)
  • • Bookings (reservations)
  • • Payments (transactions)

Key Relationships

  • • One User HAS ONE Role (1:1)
  • • One Provider PROVIDES Many Services (1:N)
  • • One User MAKES Many Bookings (1:N)
  • • One Booking HAS ONE Payment (1:1)

  • • Hotel → HotelDetails → Rooms
  • • Ride → RideDetails → Vehicles
  • • Event → EventDetails
  • • Flight → FlightDetails
